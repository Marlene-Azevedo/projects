# 📈 Plano de Evolução: Automação na Equipa

Este Roadmap descreve como pretendo ajudar a equipa a adotar o Playwright para tornar os nossos testes mais rápidos e fiáveis.

---

## 🏁 Fase 1: Organização e Aprendizagem (Curto Prazo)
**O objetivo é garantir que todos "falam a mesma língua".**

* **Documentação da Base:** Criar um guia simples de "Como correr os testes" para novos colegas.
* **Partilha de Conhecimento:** Realizar pequenas sessões de demonstração (Show & Tell) sobre como escrever cenários em Gherkin.
* **Padronização:** Garantir que todos os testes usam nomes claros e fáceis de entender por qualquer pessoa do negócio.

---

## ⚙️ Fase 2: Integração e Visibilidade (Médio Prazo)
**O objetivo é tornar os resultados dos testes úteis para todos.**

* **Testes Automáticos no GitHub:** Garantir que os testes correm sozinhos sempre que enviamos código novo (CI/CD).
* **Relatórios Visuais:** Partilhar os relatórios HTML com a equipa para que todos saibam o que falhou e porquê, sem precisar de ler código.
* **Aumento de Cobertura:** Identificar as funcionalidades mais manuais e repetitivas para as passar gradualmente para automático.

---

## 🚀 Fase 3: Estabilidade e Manutenção (Longo Prazo)
**O objetivo é manter os testes saudáveis sem gastar muito tempo.**

* **Revisão de Testes:** Verificar semanalmente se algum teste está a dar erros falsos e corrigi-lo.
* **Melhoria dos Seletores:** Substituir seletores frágeis por outros mais robustos para evitar que o teste quebre quando a página muda.
* **Feedback de Qualidade:** Usar os dados dos testes para ajudar os programadores a encontrar erros mais cedo no processo.

---

## 📊 O que ganhamos com isto? (KPIs Simples)

| Meta | O que significa? |
| :--- | :--- |
| **Menos Tempo Manual** | Libertar a equipa para testar coisas novas em vez de repetir o mesmo sempre. |
| **Confiança** | Saber que as funções principais (Login, Pesquisa) funcionam sempre. |
| **Rapidez** | Receber um OK ou Erro em poucos minutos após mudar o código. |

---

## 🛠️ Compromisso Diário
1.  **Código Limpo:** Escrever testes que sejam fáceis de ler por outros.
2.  **Apoio à Equipa:** Estar disponível para ajudar quem quer começar a usar estas ferramentas.
3.  **Melhoria Contínua:** Aprender novas funcionalidades do Playwright para otimizar o nosso trabalho.
